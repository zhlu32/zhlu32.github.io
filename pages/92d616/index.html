重定向漏洞-open redirect vuln

# 漏洞介绍

重定向漏洞(open redirect vulnerability)发生在受害者访问某URL时候，站点跳转到一个完全不同的URL，即URL的domain发生变化。

例如：假设Google有如下的URL，https://www.google.com?redirect_to=https://www.gmail.com 会让用户跳转访问Gmail。

访问上面的URL，Google服务会收到一个Http的Get请求和redirect_to的参数值，该值决定了浏览器需要重定向。访问上面的URL后，Google会返回 一个302的请求响应，浏览器会再发起一个到https://www.gmail.com的Get请求。

现在假如我们修改了上面的URL地址https://www.google.com?redirect_to=https://www.attacker.com，如果Google没有校验redirect_to 参数值的合法性，就会导致重定向漏洞，跳转到https://www.attacker.com恶意站点。

# 漏洞危害

重定向漏洞属于2013年OWASP (opens new window)前10大漏洞，但现最新的2021 OWASP前10大漏洞已不包含该漏洞。

重定向漏洞会让受害者从一个受信任站点访问到一个恶意站点，该漏洞会被用于钓鱼攻击，欺骗用户相信他提交的信息，是上传到受信任的站点，而非是恶意站点。

重定向漏洞也会被用于在恶意站点上分发恶意软件或者辅助偷取OAuth的token信息等。

重定向漏洞虽然通常不会对站点自身照成直接危害，但是会影响站点公司的声誉。

      # 实现白名单 方法2

      如果删除Parameter的方法满足不了业务需求，可以仅允许跳转白名单列表内的url。

        protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
 
   List<String> allowList = new ArrayList<String>();
   allowList.add("/profile");
   allowList.add("/contactdetails");
 
   if (auth(username, password)) {
       String redirect_url = request.getParameter("redirect_url");
       // response.sendRedirect((redirect_url != null) ? redirect_url : "/");
       if (allowList.contains(redirect_url)) {
           response.sendRedirect(redirect_url);
       } else {
           response.sendRedirect("/home");
       }
   } else {
       request.setAttribute("message", "Incorrect Credentials Supplied");
   }
 
}


        func allowed_url(url string) bool {
   switch url {
   case
       "/profile",
       "/contactdetails":
       return true
   }
   return false
}

func login(w http.ResponseWriter, r *http.Request) {
   if check_creds(r.formValue("username"), r.formValue("password")) {
       redirect = r.formValue("redirect_url")

       if allowed_url(redirect) {
           http.Redirect(w, r, redirect, http.StatusFound)
       } else {
           http.Redirect(w, r, "/home", http.StatusFound)
       }

   } else {
       http.Redirect(w, r, "/login", http.StatusFound)
   }
}

        const express = require('express');
const app = express();
const port = 3000;

app.post('/login', (req, res) => {
  const { username, password } = req.body;
  if (auth(username, password)) {
    const allowlist = ['/profile', '/contactdetails'];
    if (allowlist.indexOf(req.query.redirect_url) > -1) {
      res.redirect(req.query.redirect_url);
    } else {
      res.redirect('/home');
    }
  } else {
    res.render('login', { error: 'Incorrect Credentials Supplied' });
  }
});

app.listen(port, () => {
  console.log(`Listening on http://localhost:${port}`);
});

        # 固定Domain 方法3

        如果业务需要跳转的url是动态变化的，可以使用固定Domain的方法。

          protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
 
  if (auth(username, password)) {
      // response.sendRedirect((redirect_url != null) ? redirect_url : "/");
      String redirect = (request.getParameter("redirect_url") != null) ? request.getParameter("redirect_url") : "";
      response.sendRedirect("https://papertown.com/" + redirect);
  } else {
      request.setAttribute("message", "Incorrect Credentials Supplied");
  }

}

          func login(w http.ResponseWriter, r *http.Request) {
   if check_creds(r.formValue("username"), r.formValue("password")) {
       redirect = r.formValue("redirect_url")
      
       // http.Redirect(w, r, redirect, http.StatusFound)
       http.Redirect(w, r, "https://papertown.com/" + redirect, http.StatusFound)
   } else {
       http.Redirect(w, r, "https://papertown.com/login", http.StatusFound)
   }
}

          # 漏洞案例

          # Shopify Theme Install Open Redirect (opens new window)

          Shopify站点提供店铺装饰预览的功能，但是未对domain_name做合法性校验。

          https://app.shopify.com/themes/preview/blue?domain_name=example.com/admin ，如上的url，可导致跳转到example.com/admin，显然它的domain发生了变化。

          # Shopify Login Open Redirect (opens new window)

          Shopify站点的登录url包含的checkout_url参数值，并不是直接跳转，而是把参数值拼接到http://mystore.myshopify.com站点后面。

          http://mystore.myshopify.com/account/login?checkout_url=.attacker.com 如上的url，会跳转到http://mystore.myshopify.com.attacker.com ，显然它的domain发生了变化。

          # 总结

          从上面的实例可以发现，重定向的参数可能是容易被发现的，如redirect_to= domain_name= checkout_url=等，但有些也是很难被发现的，如r= u=等。
